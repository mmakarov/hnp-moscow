# Metadata Standards for HNP.moscow Content

## Overview
This document establishes standardized metadata practices for all content on HNP.moscow to ensure SEO optimization, consistency, and professional presentation.

## Frontmatter Standards

### Pillar Page Frontmatter Template
```yaml
---
title: "Main Topic Title - Location & Key Benefits"
description: "Comprehensive description of the service area. Include primary keywords, location, and key benefits. 15 years experience, success rate guarantee."
keywords: "primary keyword, secondary keyword, location keyword, service type"
date: YYYY-MM-DD
draft: false
type: "pillar"
---
```

### Cluster Page Frontmatter Template
```yaml
---
title: "Specific Service Title - Process, Costs & Results in Moscow"
description: "Detailed explanation of [specific service] process, requirements, timeline, and costs. 15 years experience, guaranteed results. Free consultation available."
keywords: "specific service keyword, process details, costs, timeline, Moscow"
date: YYYY-MM-DD
draft: false
---
```

## Title Standards

### Character Limits
- **Pillar Pages**: 50-60 characters
- **Cluster Pages**: 55-65 characters
- **Home Page**: 45-55 characters

### Title Structure Patterns

**Pillar Page Pattern:**
`[Service Area] - [Location] & [Key Benefit]`
Example: "Право собственности на землю в Москве – Оформление, приватизация, регистрация"

**Cluster Page Pattern:**
`[Specific Service] - [Key Details] & [Location]`
Example: "Приватизация земельного участка в Москве – Порядок, документы, сроки"

### Title Optimization Rules
- Include primary keyword at beginning
- Add location (Москва) for local SEO
- Include key benefits or differentiators
- Use proper capitalization and punctuation
- Avoid keyword stuffing

## Description Standards

### Character Limits
- **All Pages**: 150-160 characters
- **Google Display**: Optimized for search snippet display

### Description Structure Patterns

**Pillar Page Description:**
`[Service overview]. [Key statistics]. [Primary benefits]. [Call to action].`

**Cluster Page Description:**
`[Specific service details]. [Process highlights]. [Experience guarantee]. [Free consultation offer].`

### Description Optimization Rules
- Start with primary keyword
- Include location and key benefits
- Add company credentials (15 years experience)
- Include success statistics when available
- End with call to action or unique value proposition

## Keyword Standards

### Keyword Selection Guidelines
- **Primary Keyword**: 1-2 main keywords per page
- **Secondary Keywords**: 3-5 supporting keywords
- **Location Keywords**: Always include "Москва"
- **Long-tail Keywords**: Include for cluster pages

### Keyword Implementation
- Use commas to separate keywords
- Include variations and synonyms
- Avoid keyword stuffing
- Focus on user intent keywords

## Content Structure Standards

### Pillar Page Content Sections
1. **Introduction** - Service overview and company credentials
2. **Service Definition** - What the service entails
3. **Process Explanation** - Step-by-step approach
4. **Benefits & Advantages** - Why choose our services
5. **Case Studies** - Real examples and success stories
6. **Cost Structure** - Transparent pricing information
7. **FAQ Section** - Common questions and answers
8. **Call to Action** - Contact information and next steps

### Cluster Page Content Sections
1. **Service Definition** - Specific service details
2. **Process Details** - Step-by-step explanation
3. **Requirements** - Documentation and prerequisites
4. **Timeline & Costs** - Realistic expectations
5. **Common Issues** - Potential problems and solutions
6. **Success Stories** - Specific case examples
7. **FAQ Section** - Service-specific questions
8. **Next Steps** - How to get started

## Internal Linking Standards

### Pillar to Cluster Links
- Link to all cluster pages from pillar page
- Use descriptive anchor text with keywords
- Include in main content sections
- Add to "Useful Materials" section

### Cluster to Pillar Links
- Every cluster page must link back to its pillar
- Use "Main Topic Page" or similar anchor text
- Include in "Useful Materials" section

### Cross-Pillar Links
- Link related services across pillars
- Use "Related Services" sections
- Include in "Useful Materials" sections

## Content Quality Standards

### Writing Style Guidelines
- Use active voice and direct address ("you")
- Include specific numbers and statistics
- Use bullet points for lists and benefits
- Include tables for comparisons
- Add internal links to related content
- Maintain professional, authoritative tone

### Content Length Guidelines
- **Pillar Pages**: 2,000-4,000 words
- **Cluster Pages**: 1,500-2,500 words
- **Each Section**: 200-400 words
- **FAQ Sections**: 5-10 questions per page

### Visual Content Standards
- Include relevant images with alt text
- Use tables for pricing and comparisons
- Add icons for process steps when appropriate
- Ensure mobile-responsive design
