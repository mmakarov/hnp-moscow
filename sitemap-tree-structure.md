## URL Structure Breakdown

### Level 1: Domain & Home
- **Base URL**: `https://hnp.moscow/`
- **Home Page**: `/`

### Level 2: Main Service Pillars
- **Pattern**: `/{pillar-slug}/`
- **Example**: `/pravo-sobstvennosti/`

### Level 3: Service Clusters
- **Pattern**: `/{pillar-slug}/{cluster-slug}/`
- **Example**: `/pravo-sobstvennosti/privatizaciya/`

### Level 4: Geographic Targeting
- **Pattern**: `/{pillar-slug}/{geographic-location}/`
- **Moscow**: `/{pillar-slug}/moskva/`
- **District**: `/{pillar-slug}/moskva/{district-slug}/`
- **Moscow Region**: `/{pillar-slug}/moskovskaya-oblast/`


## Technical Implementation Notes

### URL Optimization
- Use hyphens (`-`) instead of underscores (`_`)
- Keep URLs under 100 characters
- Include target keywords naturally
- Use Cyrillic transliteration

### Breadcrumb Structure
Each page will implement breadcrumbs:
```
Главная > Право собственности > Приватизация
```

### Schema Markup
- **Organization**: Company information
- **LocalBusiness**: Geographic targeting
- **Service**: Each service offering
- **FAQ**: FAQ sections
- **Article**: Blog posts and resources
